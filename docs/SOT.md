# SOT — 한국외식업중앙회 경제 강의 웹 슬라이드 앱

문서 상태: v1.0 / 작성일: 2026-06-29
프로젝트명: `kfba-ai-economy-lecture`
강의 제목: **소상공인 지원 정책 및 AI를 이용한 경제교육 — AI와 경제 변화 속에서 살아남는 우리 가게**
대상: 한국외식업중앙회 소속 외식업 소상공인 / 점주 / 예비·초기 창업자
러닝타임: 60분

## 1. 반드시 만족해야 하는 산출물

1. 브라우저 전체화면 발표용 HTML/CSS/JavaScript 기반 웹 슬라이드 앱.
2. 한 파일 예제 수준이 아니라 실제 프로젝트 구조를 갖춘 정적 웹앱.
3. 강의 슬라이드와 분리된 별도 URL의 수강생 실습형 웹앱 포함. 강의자료 유출 방지를 위해 슬라이드 앱 내부에는 실습 앱을 번들링하지 않는다.
4. GitHub 레포 생성 및 무료 호스팅 방안 적용: GitHub Pages.
5. Gucci Harness 방식 적용: GitHub/SOT 중심, 주기 점검, 보고서 남김.
6. `docs/SOT.md`를 기준으로 미구현·오류·불일치 항목을 10분마다 점검하고 가능한 범위에서 코드 반영 후 보고서 갱신.

## 2. 강의 핵심 메시지

- 지원사업은 더 이상 ‘공고 하나를 신청하는 일’이 아니라 **지역성 → 브랜드 → 제품화 → 온라인 판로 → 데이터/AI 운영**으로 이어지는 사다리다.
- 정부와 지자체가 보고 싶은 것은 단순 매출 증가가 아니라 **지역 문제 해결, 자생력, 협업, 확장 가능성, 데이터 기반 운영**이다.
- 외식업의 최근 기회는 매장 장사만이 아니라 **대표 메뉴의 제품화, 온라인 판매, 예약/리뷰/재방문 데이터 축적, 로컬 콘텐츠화**에 있다.
- ChatGPT 사용법 자체보다 중요한 것은 **AI 에이전트를 점주의 반복 업무 파트너로 설계하는 법**이다.

## 4. 콘텐츠 요구사항

### 4.1 반드시 포함할 주제

- 소상공인 지원 정책의 최근 변화와 트렌드.
- 로컬크리에이터 → 강한소상공인 → 로컬브랜드/글로컬·수출형 성장처럼 이어지는 단계형 사고.
- 문화·커뮤니티 기반 창업, 지역 콘텐츠, 사업화, 제품화 흐름.
- 외식업에서 제품화 및 온라인 판매가 늘어나는 이유.
- 외식업 관련 지원사업을 찾고 실제 혜택을 볼 수 있는 탐색법.
- AI와 함께하는 외식업 시장조사 가이드.
- ChatGPT 기본 사용법보다 최신 흐름인 AI 에이전트 개념과 업무 설계.
- 홍보 문구 작성, 리뷰 관리, 블로그/SNS 활용, 무료·저비용 AI 도구 소개.

### 4.2 표현 원칙

- 소상공인 눈높이: 전문용어는 사례로 풀어 설명.
- AI스러운 과장·추상어 금지: “혁신”, “최적화”, “성장”을 남발하지 않는다.
- ‘오늘 가게에서 바로 해볼 수 있는 것’을 우선한다.
- 외식업중앙회 맥락을 반영한다: 식당/카페/분식/한식/배달/포장/예약/리뷰/원산지/위생/상권 등.

## 5. 디자인·기능 요구사항

- 16:9 고정 캔버스가 브라우저 전체화면에 맞춰 스케일링되어야 한다.
- 키보드 내비게이션: ←/→, Space, Home, End.
- 화면 하단 진행률과 슬라이드 번호 표시.
- localStorage로 현재 슬라이드 유지.
- 발표자용 슬라이드 목록/점프 기능.
- 모션은 자연스러워야 하며 `prefers-reduced-motion` 지원.
- 최소 49장 이상의 강의 슬라이드.
- 도표/표/프로세스 맵/간단한 3D 또는 입체 시각화 포함.
- 실습 웹앱은 별도 프로젝트/URL로 제공: 지원사업 매칭, AI 에이전트 설계, 리뷰 응답, 시장조사 가이드, 7일 실행표, 프롬프트 라이브러리.
- 외부 API 키나 비밀값 없이 동작해야 한다.

## 6. Gucci Harness 운영 계약

### 6.1 Source of Truth

- 본 파일 `docs/SOT.md`가 제품 요구사항의 기준이다.
- 코드 변경 시 SOT와 불일치하면 SOT 또는 코드 중 하나를 의식적으로 수정하고 `docs/qa-report.md`에 기록한다.

### 6.2 자동 점검 루프

- 10분마다 `docs/SOT.md`와 구현 상태를 비교한다.
- 점검 항목: 슬라이드 수, 내비게이션, 실습 탭, 지원사업 내용, AI 에이전트 내용, 호스팅 상태, 접근성/콘솔 오류.
- 가능한 수정은 즉시 코드에 반영하고 커밋한다.
- 수정이 위험하거나 외부 의사결정이 필요한 경우 보고서에 `HITL_REQUIRED`로 남긴다.

### 6.3 Human-in-the-loop 게이트

다음 사항은 자동으로 진행하지 않는다.

- 유료 호스팅 전환.
- 외부 서비스 API 키 입력.
- 개인정보/가게 실제 고객 데이터 입력 또는 업로드.
- 한국외식업중앙회 공식 로고/상표 사용이 필요한 디자인 확정.
- 정부 지원사업의 법적 자격요건 단정.

## 7. 검수 기준

- `npm run check` 성공.
- 로컬 서버에서 브라우저 열림.
- 콘솔 오류 0개.
- 주요 상호작용: 슬라이드 이동, 탭 전환, 입력값 기반 결과 생성, 인쇄/복사 버튼 확인.
- GitHub Pages URL 접근 가능.

## 8. 알려진 주의사항

- 지원사업은 연도와 예산에 따라 바뀐다. 슬라이드는 방향성과 탐색법 중심이며, 실제 신청 전 `소상공인24`, `소상공인시장진흥공단`, `중소벤처기업부`, 각 지자체 공고를 확인해야 한다.
- AI 도구는 개인정보·리뷰 원문·매출자료를 그대로 넣지 않는 방식으로 안내한다.

- 실습 웹앱은 강의자료와 분리된 별도 GitHub Pages 프로젝트로 배포한다.


## 10. 2026-06-29 지원사업 실전편 확장

- 강의자료에는 실제 검색으로 확인한 2025년 전후 지원사업 예시를 다수 포함한다: 강한 소상공인 성장지원, 로컬크리에이터, 스마트상점 기술보급, 온라인 플랫폼/판로·물류 지원, 희망리턴패키지 경영개선/재기사업화, 소공인 클린제조환경조성 등.
- 각 지원사업은 제목/핵심 내용/외식업 사장님 관점의 선발 전략/신청서 작성 방향을 함께 설명한다.
- 지원사업 실전편은 별도 웹앱으로도 제작해 공유 가능하게 한다.
- SOT 점검 루프는 이 요청 이후 10분 주기로 운영한다.


## 11. 2026-06-29 지역 지원사업 보강

- 중앙 단위 지원사업뿐 아니라 충남/시군 지역 지원사업 예시를 포함한다: 소상공인 사회보험료 지원, 충남 온라인 플랫폼 판로지원, 시군 경영환경개선, 음식점 입식탁자·조리장·위생 개선, 고용/인건비성 지원.
- 지역 지원사업은 외식업 사장님이 바로 체감할 수 있도록 준비 증빙, 신청서 문장, 개선 전후 사진 활용법을 포함한다.
- 지원사업 실전 웹앱은 단순 프롬프트 모음이 아니라 가게 조건 기반 추천, 증빙 체크리스트, 비용/우선순위 계산, 신청서 초안 생성까지 가능한 실전형으로 고도화한다.


## 12. 2026-06-29 중장년·시니어 친화 쉬운 지원정책 보강

- 중장년·시니어 외식업 사장님이 비교적 쉽게 확인·신청할 수 있는 지원정책을 포함한다: 전기요금/에너지 지원, 카드수수료 지원, 배달·택배비 지원, 노란우산 가입장려금, 풍수해·화재보험, 정책자금 기본 안내.

- 법적 자격요건·마감 여부는 반드시 실제 공고 확인 대상으로 안내한다.
